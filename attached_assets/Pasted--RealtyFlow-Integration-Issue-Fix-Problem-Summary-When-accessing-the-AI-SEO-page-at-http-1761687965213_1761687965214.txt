## 🔧 Required Fix

### Location: RealtyFlow Replit App

The fix needs to be made in the **RealtyFlow app** at `https://multi-users-realtyflow.replit.app`, specifically in the `/integration` endpoint handler.

### Current Code (Incorrect)
```javascript
// Somewhere in the RealtyFlow app's /integration route
app.get('/integration', (req, res) => {
  const config = {
    appUrl: 'http://localhost:5000',  // ❌ Hardcoded localhost
    iframeUrl: 'http://localhost:5000/?bypassAuth=true&...',
    authBypass: true
  };
  
  res.json({
    success: true,
    source: req.query.source,
    config: config,
    message: 'RealtyFlow integration ready'
  });
});
```

### Fixed Code (Correct)

**Option 1: Use Request Headers (Recommended)**
```javascript
app.get('/integration', (req, res) => {
  // Dynamically detect the deployment URL
  const protocol = req.protocol;  // 'https'
  const host = req.get('host');   // 'multi-users-realtyflow.replit.app'
  const appUrl = `${protocol}://${host}`;
  
  // Extract user email from query params
  const userEmail = req.query.userEmail;
  const autoLogin = req.query.autoLogin === 'true';
  const source = req.query.source;
  
  // Build iframe URL with proper authentication
  let iframeUrl = appUrl;
  if (autoLogin && userEmail) {
    iframeUrl += `/?bypassAuth=true&userId=${encodeURIComponent(userEmail)}&userType=public&agentSlug=default`;
  }
  
  const config = {
    appUrl: appUrl,  // ✅ Now uses actual deployment URL
    iframeUrl: iframeUrl,
    authBypass: autoLogin,
    userEmail: userEmail || null
  };
  
  res.json({
    success: true,
    source: source || 'unknown',
    timestamp: Date.now().toString(),
    config: config,
    message: 'RealtyFlow integration ready'
  });
});
```

**Option 2: Use Environment Variables**
```javascript
app.get('/integration', (req, res) => {
  // Use Replit environment variables or fallback
  const appUrl = process.env.REPLIT_DEPLOYMENT_URL 
    || `https://${process.env.REPL_SLUG}.${process.env.REPL_OWNER}.replit.app`
    || 'http://localhost:5000';  // Fallback for local dev
  
  const userEmail = req.query.userEmail;
  const autoLogin = req.query.autoLogin === 'true';
  
  let iframeUrl = appUrl;
  if (autoLogin && userEmail) {
    iframeUrl += `/?bypassAuth=true&userId=${encodeURIComponent(userEmail)}&userType=public`;
  }
  
  const config = {
    appUrl: appUrl,
    iframeUrl: iframeUrl,
    authBypass: autoLogin,
    userEmail: userEmail || null
  };
  
  res.json({
    success: true,
    source: req.query.source || 'unknown',
    timestamp: Date.now().toString(),
    config: config,
    message: 'RealtyFlow integration ready'
  });
});
```

## 📋 Additional Issues Found

### Issue 2: API Calls to Wrong Domain

**Console Error:**
```
GET https://realty-flow-mikebjork.replit.app/api/facebook/pages 500 (Internal Server Error)
```

**Problem:** The RealtyFlow app is making API calls to the old single-user instance (`realty-flow-mikebjork.replit.app`) instead of using its own API endpoints.

**Fix Required:**
1. Search the RealtyFlow codebase for all instances of `realty-flow-mikebjork.replit.app`
2. Replace with dynamic base URL or relative paths:
   ```javascript
   // Instead of:
   const apiUrl = 'https://realty-flow-mikebjork.replit.app/api/facebook/pages';
   
   // Use:
   const apiUrl = '/api/facebook/pages';  // Relative URL (recommended)
   // OR
   const baseUrl = window.location.origin;

   ```
