## đź“§ Email Configuration

### Development Environment

Add these to your `.env` or environment variables:

```bash
# SMTP Configuration (optional - will log to console if not set)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your-email@gmail.com
SMTP_PASS=your-app-password

# Site URL for email links
SITE_URL=http://localhost:5173
```

### Production Environment

Set these environment variables in your hosting platform:

```bash

SMTP_PORT=587
SMTP_USER=your-production-email@yourdomain.com
SMTP_PASS=your-smtp-password
SITE_URL=https://yourdomain.com
```

## đź› ď¸Ź Troubleshooting

### No Emails Received?

1. **Check console logs** - emails are logged when SMTP isn't configured
2. **Verify SMTP credentials** - test with a simple SMTP tester
3. **Check spam folder** - verification emails might be filtered
4. **Use resend feature** - both admin and public have resend buttons

### Email Links Not Working?

1. **Check SITE_URL** - must match your actual domain
2. **Verify routes** - admin uses `/verify-email`, public uses `/agent/:slug/verify-email`
3. **Check token expiry** - tokens expire in 24 hours by default

### Development vs Production Issues?

- **Development**: Uses `http://localhost:5173` by default
- **Production**: Must set correct `SITE_URL` environment variable
- **SMTP**: Can be different providers for dev vs prod

## đź¤– AI-SEO Integration - Data Passed

When users click the **AI-SEO** button, the following data is sent to the AI-SEO app:

### URL Parameters (Query String)

```
?source=nebraska-home-hub
&timestamp=1735200000000
&domain=www.imakepage.com (current host)
&userEmail=user@example.com (if logged in)
&autoLogin=true (if userEmail provided)
```

### Template Payload (via postMessage)

The app sends a `AISEO_TEMPLATE_DATA` message with real template data:

```typescript
{
  type: "AISEO_TEMPLATE_DATA",
  template: {
    // Agent Info (from template)
    agentName: template.agentName,
    agentTitle: template.agentTitle,
    companyName: template.companyName,
    companyDescription: template.companyDescription,

    // Contact Info (from template)
    phone: template.contactPhone || template.phone,
    phoneRaw: "digits only version",
    email: template.agentEmail || template.emailFromEmail,

    // Branding (from template)
    tagline: template.heroSubtitle,
    heroTitle: template.heroTitle,
    heroSubtitle: template.heroSubtitle,

    // Social Media Handles (extracted from template URLs)
    socialHandles: {
      instagram: extractHandle(template.instagramUrl),
      facebook: extractHandle(template.facebookUrl),
      linkedin: extractHandle(template.linkedinUrl),
      twitter: extractHandle(template.twitterUrl),
      tiktok: extractHandle(template.tiktokUrl),
      youtube: extractHandle(template.youtubeUrl),
    },

    // Colors (from template)
    colors: {
      primary: template.primaryColor,
      accent: template.accentColor,
      text: template.textColor,
    },

    // Website
    websiteUrl: template.externalLinkUrl,
    source: template.templateSource,
    timestamp: "ISO timestamp",
  }
}
```
